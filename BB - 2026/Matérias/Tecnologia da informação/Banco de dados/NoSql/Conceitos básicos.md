NoSQL é uma sigla para Not Only SQL, então ela suporta outras linguagens de recuperação de dados e não apenas SQL.

O NoSQL se faz necessário quando os dados não são estruturados e/ou são semi-estruturados.

O principal problema dos bancos de dados relacionais é a escalabilidade, por exemplo se tem uma tabela de usuários que aumentou muito de tamanho (e demanda) será necessário fazer um upgrade no servidor ou colocar mais um servidor.

Geralmente os Bancos de Dados NoSQL são orientados a documentos, isso significa que são capazes de manipular dados semi-estruturados (Ex: XML e JSON).

NoSQL é um termo que que funciona como um guarda-chuva para bancos de dados não-relacionais. Mas isso não significa que os bancos de dados não relacionais terão características semelhantes.

Apesar do nome sugerir o contrário, bancos de dados não-relacionais podem sim armazenas relacionamentos entre dados.

Bancos de dados não-relacionais possuem esquemas flexíveis enquanto bancos de dados relacionais possuem esquema inflexível. Bancos de dados não-relacionais tem praticamente uma ausência de esquema, o que facilita muito coisas como permitir que na tabela de Contatos a coluna de número de telefone permita um dígito a mais.

Bancos de dados não-relacionais utilizam modelos diferentes de armazenamento de dados, que podem ser divididos em quatro categorias principais: Chave-Valor, Orientado a Documentos, Orientado a Grafos e Orientado a Colunas (Colunas).
